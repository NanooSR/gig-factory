# Sell Next Steps Checklist

## Goal
Prepare these 3 gigs for a fast marketplace launch and smooth buyer handoff without changing the core gig code.

## 0) Preflight (before creating artifacts)
- [ ] Confirm 3 gig folders exist under `/gigs/` and note file paths.
- [ ] Confirm branch context is `spark/profitable-gigs-pack-1`.
- [ ] Capture one final folder listing for each gig:
  - `index.html`
  - `styles.css`
  - `script.js`
  - `README.md`
- [ ] Open each gig README and copy definitive feature statements into listings.

## 1) Packaging screenshots
- [ ] Generate screenshots for each gig in 4 viewport types:
  - Desktop 1440x900
  - Desktop 1024x768
  - Mobile 390x844
  - Tablet 768x1024
- [ ] Capture at least 3 types of state per gig:
  - Empty form state
  - Mid-entry with values filled
  - Final result/output state
- [ ] Rename files with clean naming:
  - `lead-value-roi-calculator-desktop-final-1.png`
  - `local-service-quote-calculator-mobile-output.png`
  - `website-audit-scorecard-desktop-before-after.png`
- [ ] Store screenshots in one folder per gig:
  - `docs/assets/screenshots/lead-value-roi-calculator/`
  - `docs/assets/screenshots/local-service-quote-calculator/`
  - `docs/assets/screenshots/website-audit-scorecard/`

## 2) Build zip delivery packages
- [ ] Zip each gig folder including all files shown in README:
  - `lead-value-roi-calculator-v1.zip`
  - `local-service-quote-calculator-v1.zip`
  - `website-audit-scorecard-v1.zip`
- [ ] Keep each zip root clean (no nested extra folder unless documented).
- [ ] Validate each archive by opening and clicking at least one link/file path inside.
- [ ] Store zip files in a predictable path for posting:
  - `docs/deliverables/<slug>/`

## 3) Prepare demo links
- [ ] Host lightweight demos in a temporary static test folder or existing staging path.
- [ ] Create one stable shareable URL per gig with the full folder visible.
- [ ] Run each demo on:
  - desktop width 1440
  - mobile width 390
- [ ] Confirm core calculations and CTA behavior in each demo.
- [ ] Capture final demo links in both:
  - `docs/sales-listings.md` (if needed)
  - `docs/outreach-copy.md` templates

## 4) Marketplace-ready listing setup
- [ ] Create a listing entry per gig using the exact title from `docs/sales-listings.md`.
- [ ] Add:
  - Hook sentence
  - short description
  - deliverables list
  - 3 pricing tiers with clear scope boundaries
  - FAQ section
  - 3 upsell ideas
- [ ] Include deliverable clarity: file names + included scripts/styles/readme.
- [ ] Add screenshot set and demo URL(s).
- [ ] Attach zipped package and mark included support files.
- [ ] Set pricing by value and platform norms (test low-mid-high ladder).

## 5) Outreach prep
- [ ] Finalize `docs/outreach-copy.md` with:
  - LinkedIn DM (short, non-spammy, personalized)
  - Email subject + 3-5 line body
  - 1 short social post version
- [ ] Add placeholders for buyer name, niche, and pain point.
- [ ] Keep each message ≤ 4 sentences.

## 6) Buyer handoff and close process
- [ ] Build a short close-out packet for each gig:
  - Package contents
  - Setup guide (where to upload/static host)
  - How to update branding and default values
  - Support boundary (what is and is not included)
- [ ] Send buyer an acceptance checklist:
  - screenshot of final product
  - zip delivery confirmed
  - optional demo link
  - pricing tier selected and included edits
- [ ] Collect buyer feedback after delivery and open one follow-up revision pass window.
- [ ] Save delivery evidence:
  - Delivery timestamp
  - Files delivered (zip + screenshots)
  - Demo links and copy used
  - Next-step upsell offered

## 7) Final verification before release
- [ ] Confirm only docs were modified (`docs/sales-listings.md`, `docs/outreach-copy.md`, `docs/sell-next-steps.md`).
- [ ] Verify new docs render in plain text and include all three gigs.
- [ ] Keep `docs/profit-shortlist.md` and continuity files in sync if they track launch status.
